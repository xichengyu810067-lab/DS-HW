# 網頁遊戲開發企劃書：陣列 vs 鏈結串列 (門牌分配遊戲)

## 1. 遊戲基本設定
* **遊戲名稱**：陣列 vs 鏈結串列
* **開發環境**：HTML, CSS, Vanilla JavaScript (DOM 操作)
* **遊戲目標**：透過互動式拖曳與輸入操作，讓玩家實際體驗「陣列 (Array)」與「鏈結串列 (Linked List)」在資料新增、更改、刪除、插入時的底層運作機制。
* **評分機制**：
  * 起始分數：0 分
  * 給分條件：玩家每執行正確一個指令/動作 +5 分
  * 滿分條件：100 分

## 2. 遊戲初始化流程
1. **首頁載入 (Home Page)**：
   * 顯示遊戲名稱、規則、目標與評分方式。
   * 提供一個文字輸入框供玩家「輸入名稱」。
   * 提供兩個模式選擇按鈕：「陣列模式」與「鏈結串列模式」。
2. **開局判定**：根據玩家點擊的模式，載入對應的遊戲畫面與網格系統。

---

## 3. 模式一：陣列模式 (Array Mode)

### 3.1 介面佈局與開局規則
* **網格系統**：產生 5行 × 5列 的方格（共 25 格）。
* **初始動作**：開局時，要求玩家透過點擊或輸入，在方格內「由左至右、由上至下」依序填入數字 1~15。

### 3.2 系統指派任務 (依序執行)
系統將依序下達四個任務，要求玩家手動模擬陣列的記憶體操作：
1. **新增號碼 (Append)**：
   * 系統指示：指定一個特定的空格與新號碼（不可與現有號碼重複）。
   * 玩家動作：將指定的新號碼填入該指定的空格中。
2. **更改號碼 (Modify)**：
   * 系統指示：指定已存在網格中的某個號碼，並給予替換的新號碼。
   * 玩家動作：將該格子內的舊號碼修改為新號碼。
3. **刪除號碼 (Delete)**：
   * 系統指示：指定一個要刪除的目標號碼。
   * 玩家動作：先將目標號碼從格子中刪除，接著必須將該號碼後方**所有的號碼往左拖移一格**，以填補空缺（模擬陣列連續記憶體位移）。
4. **插入號碼 (Insert)**：
   * 系統指示：在現有的 1~15 號資料區間內，隨機指定一個目標格子，並給予一個待插入的新號碼。
   * 玩家動作：先將該目標格子及其後方**所有的號碼往右拖移一格**騰出空間，接著將新號碼填入該目標格子中。

## 4. 模式二：鏈結串列模式 (Linked List Mode)

### 4.1 介面佈局與開局規則
* **網格系統**：產生 10行 × 10列 的方格（共 100 格）。
  * X 軸：從左至右依序為第 1 行至第 10 行。
  * Y 軸：從上至下依序為第 1 列至第 10 列。
* **初始動作**：
  * 系統隨機選擇 30 個空格，每兩格為一組，共生成 15 組號碼（即 15 個 Node）。
  * **節點結構**：每組的「左邊格子」顯示**本格號碼 (Data)**，「右邊格子」顯示**下個號碼所在的行列座標 (Pointer / Next)**。

### 4.2 系統指派任務 (依序執行)
系統將依序下達四個任務，要求玩家手動模擬鏈結串列的指標操作：
1. **新增號碼 (Append)**：
   * 系統指示：指定地圖上一處空地（兩格一組）與新號碼（不可重複）。
   * 玩家動作：在指定空地的左格填寫新號碼，並將目前串列最後一個節點的右格（指標）更改為指向這個新號碼的行列位置。
2. **更改號碼 (Modify)**：
   * 系統指示：隨機指定一個現有的號碼，要求更改為新號碼。
   * 玩家動作：直接將該節點左格的號碼變更為新號碼（指標不變）。
3. **刪除號碼 (Delete)**：
   * 系統指示：指定一個要刪除的目標號碼。
   * 玩家動作：清空該目標號碼的兩格內容，並將「前一個節點的右格（指標）」更改為指向「被刪除節點的下一個節點座標」（模擬 Bypass 動作，不需拖移資料）。
4. **插入號碼 (Insert)**：
   * 系統指示：指定將新號碼插入於特定節點 A 與節點 B 之間，並指定一處空地安置新號碼（節點 C）。
   * 玩家動作：在空地的左格填入新號碼（節點 C），將節點 A 的右格指向節點 C 的座標，並將節點 C 的右格指向節點 B 的座標。
